Kiến trúc: LINE / WhatsApp → Craveva AI → Hub

Người dùng chat trên LINE hoặc WhatsApp → webhook tới https://ai.craveva.com/ → AI xử lý và truy vấn dữ liệu từ hệ thống http://hub.craveva.com/ (sản phẩm, giá, …) → trả lời trong chat.
Bản multi-tenant (1 tài khoản AI + 1 company Hub / khách): chat_ai_hub_mt.html.

1. Sơ đồ thành phần (luồng dữ liệu)

flowchart TB subgraph channels["Kênh chat"] direction TB U["Người dùng"] subgraph messaging["LINE / WhatsApp"] direction TB LINE["LINE Official Account"] WA["WhatsApp Business"] end U --> LINE U --> WA end subgraph systems["Nền tảng AI & Craveva Hub"] direction LR subgraph ai_layer["Nền tảng AI"] direction TB AI["Craveva AI\nhttps://ai.craveva.com\nWebhook + Agent / LLM"] OUT["Trả lời chat\nqua API LINE / WhatsApp"] AI --> OUT end subgraph hub_layer["Craveva Hub"] direction TB HUB["hub.craveva.com\n(ứng dụng Hub)"] DB[("Database Hub\nsản phẩm, giá, …")] HUB --> DB end AI -->|"Truy vấn (API Hub / theo triển khai)"| HUB AI -.->|"Tuỳ cấu hình:\nđọc DB trực tiếp"| DB end LINE -->|"HTTPS Webhook"| AI WA -->|"HTTPS Webhook"| AI
Ghi chú triển khai: Cạnh nét đứt tới Database là tùy chọn khi dịch vụ AI được phép kết nối DB (hoặc replica) trực tiếp. Nhiều triển khai chỉ dùng API Hub làm ranh giới bảo mật. Bố cục trên → dưới: Kênh chat luôn phía trên; hàng dưới là AI (có nhánh Trả lời chat qua API nền tảng) và Hub/DB cạnh nhau. Không vẽ mũi tên AI → lại LINE/WA vì tạo vòng khiến Mermaid đảo khung (chat bị tụt xuống). Luồng 2 chiều đầy đủ xem mục 2 (sequence bên dưới).

2. Trình tự: hỏi sản phẩm / giá trong chat

sequenceDiagram autonumber actor U as Người dùng participant C as LINE / WhatsApp participant AI as Craveva AI (ai.craveva.com) participant H as Hub (hub.craveva.com + DB) U->>C: Tin nhắn (giá, SKU, mô tả SP, …) C->>AI: Webhook HTTPS (payload tin nhắn / ngữ cảnh) activate AI AI->>AI: Agent / LLM phân tích ý định AI->>H: Truy vấn sản phẩm, giá, tồn kho, … activate H H-->>AI: Bản ghi / API response deactivate H AI-->>C: Trả lời (text, rich message, …) deactivate AI C-->>U: Hiển thị trong cuộc hội thoại

3. Điểm cần lưu ý (vận hành & bảo mật)

  • Webhook: URL công khai HTTPS, xác thực chữ ký / secret (LINE, Meta).
  • Hub DB: firewall / allowlist IP nếu AI kết nối trực tiếp tới Cloud SQL.
  • Tách staging vs production cho ai.craveva.com và hub.craveva.com.
