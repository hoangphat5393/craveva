Kiến trúc multi-tenant: LINE / WhatsApp → Craveva AI → Hub

Mỗi khách hàng doanh nghiệp có 1 tài khoản trên https://ai.craveva.com và 1 company trên hub.craveva.com (dữ liệu ERP theo company_id). Người chat trên LINE/WhatsApp là người dùng cuối, khác với tenant admin cấu hình tích hợp. Chart dùng Cách 1 (CSS font 18px); vùng Provisioning xanh dương, Runtime xanh lá.

So sánh nhanh: Bản đơn giản (một AI, một Hub, một DB) — chat_ai_hub.html. File này bổ sung provisioning, map tenant, gateway DB và kênh chat theo từng khách.

4. Điểm vận hành & bảo mật

  • Tách tenant: Mọi truy vấn runtime phải gắn đúng company_id = N (gateway VIEW hoặc API + secret).
  • Webhook: URL/secret LINE hoặc Meta theo channel; AI route về đúng workspace.
  • Cloud SQL: Allowlist egress IP production AI (136.110.35.154/32) nếu dùng public IP.
  • Staging vs production: Tách ai, hub, DB và credential — không dùng chung gateway user.
  • Ghi đơn: Ưu tiên API Hub (AI_ORDER_INTEGRATION_REST) thay vì INSERT trực tiếp DB ERP.
