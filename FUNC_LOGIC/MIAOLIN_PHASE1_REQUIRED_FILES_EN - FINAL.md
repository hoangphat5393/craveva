## Miaolin Foods – Phase 1 Required Files (For Craveva B2B AI Smart Distribution Platform)

Below is the list of **data files and templates** required from the Miaolin team to fully enable **Phase 1** of the Miaolin Foods B2B AI Smart Distribution Platform, based on:

- The **system implementation** already completed (imports, DB structure, custom fields), and
- The **signed contract** + proposal (daily file‑based integration with Digiwin ERP and AI modules).

---

### 1. Customer Master

**File name:** `Miaolin Customer.xlsx` (FINAL version)  
**Purpose:** Import all B2B customers into the Craveva platform.

**Requirements:**

- One row per customer.
- Please make sure the following columns are filled (at minimum):
    - `客戶代號 | Customer Code`
    - `客戶簡稱 | Customer Short Name`
    - `送貨地址 | Delivery Address`
    - `TEL_NO(一)`, `TEL_NO(二)` | Phone 1, Phone 2
    - `統一編號 | Tax ID`
- Business / segmentation columns (already supported in the system as custom fields):
    - `業務員 | Salesperson`
    - `部門 | Department`
    - `業務助理名稱 | Sales Assistant Name`
    - `通路別 | Channel Type`
    - `型態別 | Business Type`
    - `最近交易 | Last Transaction Date`
    - `交易條件 | Payment Terms`
    - `歇業日期 | Business Closure Date`
- Optional but recommended:
    - `客戶(集團)分級 | Customer Grade`
        - If you plan to use this for reporting/segmentation, we will create a custom field `customer_grade` and import this column.

---

### 2. Product Master

**File name:** `Miaolin Product.xlsx` (FINAL version)  
**Purpose:** Import all SKUs (including unit/box variations) into the Craveva product database.

**Requirements:**

- One row per SKU.
- Please make sure all current/live SKUs are included.
- Required columns (already mapped to the system database):
    - `品號 | SKU`
    - `品名 | Product Name`
    - `規格 | Specification`
    - `庫存單位 | Unit Type` (e.g. 包, KG)
    - `商品級別 | Product Grade`
    - `商品來源 | Product Origin`
    - `品牌類別 | Brand`
    - `保存天數 | Shelf Life (days)`
    - `備貨型態 | Inventory Type`
    - `儲存溫層 | Storage Condition`
    - `標準價 | Standard Price`
    - `成箱價 | Price per Box` (if applicable)

---

### 3. Digiwin ERP Standard Import Template

**File name:** `Digiwin_Standard_Import_Template.xlsx`  
**Purpose:** Define the standard format required by Digiwin ERP for importing orders, so we can build the **Virtual SKU Mapper** and daily export from Craveva to Digiwin.

**Requirements:**

- This should be the **official import template** used by Digiwin ERP (Excel/CSV).
- Please clearly mark which columns are **mandatory** for Digiwin to accept an import, for example:
    - Client Code
    - Inventory No. (ERP SKU / Product Code)
    - Warehouse Code
    - Quantity
    - Unit
    - Any other mandatory fields required by Digiwin.

---

### 4. Pricing Table from Digiwin (Client-Specific Pricing)

**File name:** `Digiwin_Client_Price_Table.xlsx`  
**Purpose:** Support the **5-Level Pricing Engine**, especially Level 4 **Client-Specific Contract Price**, via daily Excel import from Digiwin to Craveva.

**Requirements:**

- One row per (Client + SKU + Price) combination.
- Recommended minimum columns:
    - `Client Code`
    - `Product SKU` (Inventory No.)
    - `Contract Price` (client-specific price for that SKU)
- Optional (if already available in Digiwin exports):
    - Currency
    - Effective Date
    - Expiry Date
    - Price type (if multiple price lists are used)

---

### 5. Inventory Export from Digiwin (Daily Morning Sync)

**File name:** `Digiwin_Inventory_Export.xlsx`  
**Purpose:** Update **sellable inventory** in Craveva every morning, following the “Daily File Exchange” model defined in the contract.

**Requirements:**

- One row per (Warehouse + SKU).
- Recommended minimum columns:
    - `Product SKU` / `Inventory No.`
    - `Warehouse Code`
    - `Quantity On Hand` (or equivalent inventory quantity field)
- Optional (if available and relevant to Miaolin’s operations):
    - `Batch Number`
    - `Expiry Date`

---

### 6. Order Status Export from Digiwin (Status & Inventory Alignment)

**File name:** `Digiwin_Order_Status_Export.xlsx`  
**Purpose:** Keep Craveva order status in sync with Digiwin, so that (as described in the contract):

- Customers can check their order status in real-time.
- AI Procurement Assistant can answer “where is my order?”.
- Inventory can be reconciled against confirmed / delivered / cancelled quantities.

**Requirements:**

- One row per order (or per order line, depending on Digiwin’s export format).
- Recommended minimum columns:
    - `Order ID` (Digiwin order number)
    - `Fulfillment Status` (e.g. Confirmed / Picking / Delivered / Cancelled)
    - `Delivery Date & Time` (if available)

---

### 7. Daily Order Summary from Craveva to Digiwin

> **Note:** This file is **generated by Craveva**, but its structure is constrained by Digiwin’s standard import template. It is listed here to make the end‑to‑end daily loop explicit.

**File name:** `Craveva_Daily_Order_Summary_for_Digiwin.xlsx`  
**Purpose:** Send all Craveva orders back to Digiwin for accounting and fulfillment, using the Virtual SKU Mapper to convert sales units (bags/units) into Digiwin inventory units (cartons/boxes).

**Requirements / Alignment:**

- Format must follow **`Digiwin_Standard_Import_Template.xlsx`**:
    - `Client Code`
    - `Inventory No. / Product SKU` (already converted to Digiwin’s carton/unit)
    - `Warehouse Code`
    - `Quantity`
    - Any other **mandatory** fields Digiwin requires for import (see section 3).
- Miaolin’s operations team will:
    - Import this file into Digiwin every night / next morning.
    - Define in Digiwin how to handle duplicate or erroneous imports (as mentioned in the contract).

---

### 8. AI Training Data

#### 7.1. Line / Text Order Chat Samples

**File name:** `Line_Order_Chat_Samples.zip`  
**Purpose:** Train the **AI Procurement Assistant** to understand real-world ordering language used by Miaolin’s B2B clients on Line and text.

**Requirements:**

- Around **50–100 real order conversations** (anonymized).
- Can be:
    - Screenshots, or
    - Exported text / chat logs.
- Please include a variety of scenarios, such as:
    - “Same as last week”
    - “Add 2 cartons of butter”
    - “The usual flour”
    - Mixed language orders (Chinese + English) if applicable.

#### 7.2. Product Knowledge Base (Brand Specs)

**Folder name:** `Brand_Product_Specs/`  
**Purpose:** Train the **AI Sales Agent** to answer detailed product questions (fat content, origin, usage, storage, etc.) for all Miaolin agency brands.

**Requirements:**

- A folder containing product specification documents for each major brand, for example:
    - Product catalog PDFs
    - Technical spec sheets
    - Any official product documentation used by your sales team.
- The more complete this folder is, the more accurate and useful the AI Sales Agent will be.

---

If you have any questions about the formats above, we can review one sample export for each file together and then finalize the templates for daily operations.
